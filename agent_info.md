## Agent Roles Overview

# Agent 1 — Interview Analyst

Takes the raw Teams transcript and extracts meaning from noise.

Identifies the core problem
Pulls out user goals and pain points
Captures key insights in a structured summary

Think: “What did the user actually mean?”

# Agent 2 — Product Owner

Transforms the structured insights into a clear, usable requirement.

Writes a user story
Defines acceptance criteria
Ensures the requirement is testable and valuable

Think: “What should we build?”

# Agent 3 — Technical Analyst

Breaks the requirement into implementation-ready work.

Generates granular tasks
Produces BDD / Gherkin scenarios
Clarifies expected system behaviour

Think: “How do we build it?”

# Agent 4 — Delivery Tracker

Persists everything into a structured dataset (Excel).

Stores tasks as rows
Links each task to its parent user story
Enables tracking, reporting, and search

Think: “How do we manage and track it?”
Note - this is a deterministic util, not an AI agent
